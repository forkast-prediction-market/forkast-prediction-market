## Endpoint

`GET /v1/markets/{marketId}`

## Path parameters

Provide the unique market identifier returned from the listing endpoint.

## Response structure

The payload returns market metadata, order book snapshots, resolution details, and your account's position if authenticated.

<Callout type="info" title="Cache smart">
  Responses from {process.env.NEXT_PUBLIC_SITE_NAME} are edge cached for a few
  seconds. Use the <code>ETag</code> header to avoid burning through rate limits
  when polling.
</Callout>

<Accordions type="multiple" defaultValue={['metadata']}>
  <Accordion title="Metadata" value="metadata">
    Provides identifiers, question text, category tags, and timestamps needed to
    render the market directory or catalog.
  </Accordion>
  <Accordion title="Order book" value="orderbook">
    Includes bid and ask depth, mid price, and liquidity snapshots so you can
    sync the trading interface programmatically.
  </Accordion>
  <Accordion title="Resolution" value="resolution">
    Returns status transitions, selected outcome, and oracle evidence whenever a
    market is resolving or already closed.
  </Accordion>
  <Accordion title="Your position" value="position">
    When authorized, appends the account's exposure, average entry price, and
    unrealized PnL for the requested outcome.
  </Accordion>
</Accordions>
