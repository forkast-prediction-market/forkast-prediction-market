## Endpoint

`POST /v1/orders`

## Request body

Include the market identifier, outcome ID, side (`buy` or `sell`), order type, size, and optional limit price.

<Tabs items={['REST', 'Node SDK']}>
  <Tab>

```http
POST /v1/orders HTTP/1.1
Host: api.forkast.market
Authorization: Bearer YOUR_KEY
Content-Type: application/json

{
  "marketId": "elections-2024",
  "outcomeId": "yes",
  "side": "buy",
  "orderType": "limit",
  "size": "150",
  "price": "0.54"
}
```

  </Tab>
  <Tab>

```ts
import { createTradingClient } from '@forkast/sdk'

const client = createTradingClient({
  apiKey: process.env.FORKAST_API_KEY!,
})

await client.placeOrder({
  marketId: 'elections-2024',
  outcomeId: 'yes',
  side: 'buy',
  orderType: 'limit',
  size: '150',
  price: '0.54',
})
```

  </Tab>
</Tabs>

## Response

Successful submissions return an order ID that you can track for fills or cancellation.

<Callout type="info" title="Tip">
  Client-side orders submitted from {process.env.NEXT_PUBLIC_SITE_NAME} use the same
  payload schema, so reusing this contract keeps integrations compatible with
  future platform upgrades.
</Callout>
